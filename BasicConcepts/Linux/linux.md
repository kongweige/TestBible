### 性能分析
**CPU**

查看cpu信息
```shell    
cat /proc/cpuinfo
```  
查看系统负载和CPU利用率
```shell
top
```
ps命令的cpu利用率是平均的，不适用于与性能分析
```shell
ps
```

**内存**：总的内存大小，是否有可用内存，进程申请内存时是否有受到限制，申请了多少内存，内存的使用是否合理，有没有内存泄漏
* 系统级别
  ```shell
  查看系统的内存情况
  free 
  meminfo 保存了更多的内存使用数据
  cat /proc/meminfo
  ```
* 进程级别
  ```shell
  top

  查看进程列表快照（快照：某个特定时刻的系统状态的静态副本）
  ps aux
  ```

**网络**：链接数不够（系统级别的一些参数会导致没法建立更多的连接） 
统计网络连接情况
```shell
netstat -t：显示TCP协议的连接信息
netstat -u：显示UDP协议的连接信息
netstat -n：以数字形式显示地址和端口，而不进行域名解析

netstat -p：显示每个网络连接对应的进程PID和程序名称
netstat -l：仅显示侦听状态的连接。

netstat -tnlp


ulimit： 命令来限制其所能使用的系统资源，如最大打开文件数、最大内存使用量等
core file size：指定核心转储文件的最大大小。通常设为0表示禁止生成核心转储文件
data seg size：指定数据段的最大大小。unlimited表示没有限制
file size：指定单个文件的最大大小。unlimited表示没有限制
max memory size：指定最大内存大小。unlimited表示没有限制
open files：指定同时打开的文件描述符的最大数量

ulimit -a

```

### 统计命令
排序：sort
```shell
-b：忽略开头的空白字符
-f：将小写字母看做大写字母
-h：根据存储容量排序（KB、MB、GB）
-n：按数字排序
-o：把结果写入文件
-r：以相反的顺序来排序
-t：指定分隔符，默认为空格
-V：按照数字版本排序
-k：指定排序的关键字，与-t 参数配合使用
```
去除重复：uniq
```shell
-c：统计重复出现的次数
-d：所有邻近的重复行只被打印一次。重复次数要大于等于 2
-D：所有邻近的重复行将全部打印
-f：跳过对前 n 个列的比较
-s：跳过对前 n 个字符的比较
-w：只对每行前 n 个字符进行比较
```
字符统计：wc
```shell
-c：统计字节数：chars
-l：统计行数
-w：统计但单词数
-L：打印最长行的长度
```

## grep
功能：grep 用于在文本文件中搜索匹配特定模式的行，并将匹配的行打印出来

* -i（忽略大小写）
* -r（递归搜索）
* -n（显示行号）
* -v（显示不匹配行）
* -E（正则）
* -o（仅显示匹配到的文本，而不是整行）
